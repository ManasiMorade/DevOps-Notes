# Day 1 - DevOps & Agile

## DevOps
*Definition:*
DevOps is a culture and set of practices that helps Development and Operations teams work together to deliver software quickly, safely, and continuously.

*Purpose:*
- Build
- Test
- Release
- Monitor
- Maintain

*Tools:*
- Git
- GitHub
- Docker
- Jenkins
- Datadog

## Agile Methodology

*Definition:*
Agile is a software development approach where work is divided into small parts called *Sprints* (2–4 weeks).

*Why Agile?*
Traditional Waterfall model is rigid. Requirement changes are difficult and costly. Agile provides flexibility.

### Agile Lifecycle
Requirements
↓
Planning
↓
Sprint (2–4 Weeks)
↓
Development
↓
Testing
↓
Review
↓
Feedback & Improvement
↓
Next Sprint

### Key Features of Agile
- Iterative Development
- Continuous Customer Feedback
- Flexibility
- Continuous Delivery
- Team Collaboration

### Agile Roles

*Product Owner*
- Represents customer requirements
- Prioritizes features

*Scrum Master*
- Ensures Agile practices are followed

*Development Team*
- Design, Development and Testing

### Advantages of Agile
- Faster Delivery
- Better Customer Satisfaction
- Easy Requirement Changes
- Improved Quality
- Reduced Risk
- Continuous Feedback

### Disadvantages of Agile
- Requires Active Customer Involvement
- Difficult for Very Large Projects
- Scope Changes Frequently
- Requires Experienced Team Members

### Agile vs Waterfall

| Agile | Waterfall |
|---------|---------|
| Flexible | Rigid |
| Changes Allowed Anytime | Changes Difficult |
| Continuous Delivery | Delivery at End |
| Regular Customer Involvement | Customer Involvement Mainly at Start & End |

1. Scrum
